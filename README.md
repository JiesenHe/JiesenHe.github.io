# VigorMall

# 规范

## 测试规范

请使用`尺寸：iPone12 Pro`进行测试，避免不同设备尺寸导致的问题

## 组件使用

- 组件不需要使用`import`导入，直接使用代码即可
- script请使用ts语法书写，防止冲突

## 文件修改规范

禁止随意修改他人负责部分文件

## 上传规范

`updata <时间：24/12/06-16:50> <上传人: JiesenHe> <修改内容（可选项）>`

示例`updata 24/12/06-16:50 JiesenHe updata README`

## 文件建立规范

- 参考初始文件结构创建文件夹
- components文件夹下多次服用除多次复用组件，其余组件放置于`components`文件夹下的单独文件夹下


## 书写规范

- 使用响应式API构建Vue代码
- 使用`setup`选项构建组件，已安装name插件，在setup中直接书写`name`

## 命名规范

- 使用大驼峰命名法
- 页面后缀为View
- 变量遵循小驼峰命名法
